Event Management API

FastAPI-based REST API for managing events on AWS

Overview

This API provides a complete CRUD interface for managing events. It's built with FastAPI and deployed on AWS using serverless architecture (Lambda + API Gateway + DynamoDB).

Base URL

https://ivtbuugyh1.execute-api.us-west-2.amazonaws.com/prod/

Features

  • Create, Read, Update, and Delete events
  • Filter events by status
  • Comprehensive input validation
  • CORS enabled for web access
  • Proper error handling with HTTP status codes

Event Schema

Each event contains the following properties:

eventId (string)

Unique identifier for the event. Can be provided during creation or auto-generated.

title (string, required)

Event title. Length: 1-200 characters.

description (string, required)

Event description. Length: 1-2000 characters.

date (string, required)

Event date in ISO format. Examples: "2024-12-25T10:00:00Z" or "2024-12-25"

location (string, required)

Event location. Length: 1-200 characters.

capacity (integer, required)

Maximum number of attendees. Range: 1-100000.

organizer (string, required)

Event organizer name. Length: 1-200 characters.

status (string, required)

Event status. Allowed values: draft, published, cancelled, completed, active

API Endpoints

GET /events

Description: Retrieve a list of all events.

Query Parameters:

  • status (optional) - Filter events by status (e.g., ?status=active)

Response: 200 OK

[
  {
    "eventId": "123e4567-e89b-12d3-a456-426614174000",
    "title": "Tech Conference 2024",
    "description": "Annual technology conference",
    "date": "2024-12-25T10:00:00Z",
    "location": "Tokyo Convention Center",
    "capacity": 500,
    "organizer": "Tech Events Inc",
    "status": "published"
  }
]

POST /events

Description: Create a new event.

Request Body:

{
  "eventId": "optional-custom-id",
  "title": "Tech Conference 2024",
  "description": "Annual technology conference",
  "date": "2024-12-25T10:00:00Z",
  "location": "Tokyo Convention Center",
  "capacity": 500,
  "organizer": "Tech Events Inc",
  "status": "published"
}

Response: 201 Created

Note: The eventId field is optional. If not provided, a UUID will be generated automatically.

GET /events/{eventId}

Description: Retrieve a specific event by ID.

Path Parameters:

  • eventId - The unique identifier of the event

Responses:

  • 200 OK - Event found and returned
  • 404 Not Found - Event does not exist

PUT /events/{eventId}

Description: Update an existing event.

Path Parameters:

  • eventId - The unique identifier of the event

Request Body: (all fields optional)

{
  "title": "Updated Conference Title",
  "capacity": 600
}

Responses:

  • 200 OK - Event updated successfully
  • 404 Not Found - Event does not exist
  • 400 Bad Request - No fields to update

DELETE /events/{eventId}

Description: Delete an event.

Path Parameters:

  • eventId - The unique identifier of the event

Responses:

  • 204 No Content - Event deleted successfully
  • 404 Not Found - Event does not exist

Error Responses

Status Code Description
400 Bad Request - Invalid input data or validation error
404 Not Found - The requested resource does not exist
500 Internal Server Error - An unexpected error occurred

Error Response Format

{
  "detail": "Error message describing what went wrong"
}

Module Documentation

main.py

The main application module containing the FastAPI application and all API endpoints.

Classes

EventBase

Base Pydantic model for event data with validation rules.

Validators:

  • validate_date - Validates date format (ISO or YYYY-MM-DD)
EventCreate

Model for creating new events. Extends EventBase with optional eventId field.

EventUpdate

Model for updating events. All fields are optional.

Event

Complete event model including eventId. Used for API responses.

Functions

create_event(event: EventCreate)

Creates a new event in DynamoDB. Generates UUID if eventId not provided.

Returns: Event object with 201 status

list_events(status: Optional[str])

Lists all events, optionally filtered by status parameter.

Returns: List of Event objects

get_event(event_id: str)

Retrieves a single event by ID.

Returns: Event object or 404 error

update_event(event_id: str, event_update: EventUpdate)

Updates an existing event with provided fields.

Returns: Updated Event object or 404 error

delete_event(event_id: str)

Deletes an event by ID.

Returns: 204 No Content or 404 error

Configuration

  • CORS: Enabled for all origins, methods, and headers
  • DynamoDB Table: Configured via EVENTS_TABLE_NAME environment variable
  • Lambda Handler: Mangum adapter for AWS Lambda integration

Deployment

This API is deployed on AWS using the following services:

  • AWS Lambda: Runs the FastAPI application
  • API Gateway: Provides HTTP endpoints and routing
  • DynamoDB: NoSQL database for event storage

Deployment is managed using AWS CDK. See the infrastructure/ directory for configuration details.
